The pace of model development is relentless. The best option for a given use case changes constantly, and no single model wins everywhere. One excels at structured analysis. Another at creative synthesis. Another at code generation. The right answer shifts every few weeks.

I've decoupled my knowledge from any single model. Switching is painless when the context layer is mine. Frontier model, local model, offline model: the context doesn't care where it runs.

I will switch models this year, probably more than once. Owning the context layer means every model switch becomes an upgrade, not a restart.
