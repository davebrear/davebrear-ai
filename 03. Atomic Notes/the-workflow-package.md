I stopped evaluating models in isolation. I build what I call a workflow package: skill + prompt + context.

The skill defines what to do. The prompt defines how to instruct. The [[the-context-layer|context]] defines what to know.

I run the same workflow package against each new model as it drops. Benchmark model + workflow together, not model alone. When I own the workflow package, every new model release becomes a simple swap-and-compare. That's only possible if the components are mine. [[model-portability|Model portability]] means owning not just the context but the entire evaluation harness. [[context-is-the-differentiator|Context is the differentiator]], but the workflow package is what turns context into capability.
