---
title: "Organise by Description, Not Location"
type: atom
tags:
  - knowledge-management
---

I stopped creating a folder for everything. I describe what things are, not where they sit. A single note can belong to multiple contexts when the description is right.

This is the [[zettelkasten-as-thinking-system|Zettelkasten]] insight applied to the AI age: categories are rigid, descriptions are flexible. A note about "enterprise security concerns around AI adoption" can live in any folder, but if its metadata and content describe it well, every [[four-modalities-of-search|search modality]] can find it regardless of where it sits.

Folders are locations. Metadata and links are descriptions. In a network, description wins. [[let-structure-emerge|Let structure emerge]] from the descriptions rather than imposing it upfront.
