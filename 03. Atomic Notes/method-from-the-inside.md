When you ask the AI to describe a method abstractly, the first proposal often reads as generic spec language. Aspirational, plausible, applicable to almost any methodology. Could have come from any blog post.

The fix is to ask it to describe the method from the inside of a session running that method. Not "describe Phase 2" but "describe Phase 2 from what is happening in this Phase 2, right now."

The rewrite is grounded in observed behaviour rather than aspiration. The descriptive verbs are the tell - rename, merge, split, reorder, drop, cut. Verbs that come from the act, not from theorising about the act.

This is meta-recursion as a craft move. The method is being described while it is being run. The description gets honest because it cannot afford to drift.
